statika

what

a library for working with machine configurations (now) and systems specifications/resources (future).

a lot of work → Alexey Alekhin

how

  • static
  • type-level
  • dependently-typed
  • configuration = code

advanced stuff, we’re pushing the Scala type system to its limits.

But also simple! essentially,

  • dependent record types

concepts

  • Bundles get applied to instances
  • Distributions they apply Bundles

bundles

A Bundle has

  • a version
  • dependencies → Bundles
  • an install method
// just the idea
trait Bundle {

  val version: Version
  type Deps <: HList: OfBundles
  val deps: Deps
  def install[D <: Dist](d: D): Results
}

versions

Every bundle needs to be compiled, tested and then published. When you depend on it, you’re depending on that particular version.

// example of bundle versioning

dependencies

are explicit, and are Bundles too; you can use them for writing your install method.

object S3cmd extends 
  Bundle(Git :: Python :: HNil)

install

but not only that! think of it as just code that you want to be run in your instance.

distributions

They have

  • a list of members
  • which are those bundles
  • which they can apply
// just the idea
// dists are Bundles too!
trait Dist extends Bundle {

  type Members <: HList
  val members: Members
  // code for applying members etc
}
