the future

abstract over execution

Identify a semantic universe \(\mathcal{U}\) which represents the systems that you want to build. Describe your system \(S\colon \mathcal{U}\) in that framework

abstract over execution

Now an execution is just

\[ E\colon S \to P \]

where \(P \colon \mathcal{U}\). RFC Walters and others

model platform state

explicitly: if you want to execute something on AWS, express what is out of control also in \(\mathcal{U}\):

\[\mathrm{amz} \colon \mathcal{U}\]

correct by construction

Now, what you have is a family of executions parameterized by AWS state

\[ E\colon S \times \mathrm{amz} \to P \times \mathrm{amz} \]

higher-order state machines!!

possible now?

Not really. In this direction → Futures, Dataflow concurrency, …

But, we’re working on it!
